你还漏了的 4 件事
1. Prompt 设计（这是最关键的）
不是随便写一句话就完了。要保证 Claude 每次都返回一致的 JSON 结构，Prompt 必须精确：


你需要定下来的内容：
├── 系统角色设定（"你是一个资深代码审查专家..."）
├── 四个维度的具体定义（什么叫安全问题？什么叫性能问题？给出示例）
├── 输出 JSON Schema 的精确格式（字段名、类型、是否必填）
├── 严重程度的判定标准（high/medium/low 分别对应什么情况）
└── 不审查的情况（空代码、非代码文本，返回什么）
2. 结构化输出的实现方式
Claude API 有两种方式拿到结构化 JSON：

方式	优点	缺点
Prompt 里要求返回 JSON	简单，直接写在 prompt 里	Claude 偶尔会多输出解释文字
Tool Use / JSON Mode	强制返回合法 JSON	需要定义 schema
你该选的： 先用 Prompt 方式（简单），如果发现返回格式不稳定再换 Tool Use。

但无论哪种，你的后端必须做一件事：校验 JSON。Claude 返回的内容可能带 markdown 包裹（```json ... ```），你的代码要能剥离它再解析。
